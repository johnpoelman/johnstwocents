Yes, it is possible. You can run a website for virtually no money thanks to a couple of software and services. Actually, you don’t need a full fledged CMS for a personal website.

A CMS is the abbreviation of a content management system and is a website that’s built around a database and forms to add content to the website. This means a lot of code that can be abused by not so friendly visitors. Most of the times the CMS software is free software but you need decent hosting to run it. Perhaps you have heard from Wordpress or Drupal to name a few.

How do I run my website?

After reading the above I guess you will assume that I don’t run a dynamic CMS as the backend for my personal website. You are correct!

Let me run through everything I use to run this website almost for free. I do pay less then 10 dollar each year to keep this website alive. Not bad you say? I don’t think so too!

Domain and hosting services

  • Cloudflare
  • Github pages

Cloudflare

Cloudflare does a couple of things for me. Perhaps the most important thing of having a website is making it accessible in a convenient way thus you need to have your own domain. I promoted Cloudflare to be my domain registrar and I ordered a dotcom domain for less then 10 dollar.

By doing this it’s very easy to take advantage of all the features of Cloudflare such as DNS, SSL certificates, their proxy services and tunnels. I’m not going to describe them all in detail, that is perhaps something for another post.

Github Pages

Github Pages is a feature you can enable on a GitHub repository to use it to serve it’s content as a static website. By that I mean that it only serves static HTML files. Did you know it’s safer to serve HTML files rather then using a dynamic website? On a static website there is literally no way in!

Content creation services

  • Hugo
  • Some Hugo theme

Hugo, a static website generator

Hugo is a free and open source tool to generate static HTML pages from your local content written in Markdown. That is a lot of information in just one sentence but it covers everything that’s to it, really!

Hugo is a software that is running on your computer, you use it to:

  • Create new posts
  • Publish posts
  • Build your website pages

When you tell Hugo to build your site it will do so and save all the files in the public folder by default. Upload the files in the public folder to your Github repo and you are golden.

Some Hugo theme

Hugo relies heavily on your theme to build up it’s pages. You can choose one from the many that are available from the marketplace or you can create your own. I’ve chosen the M10c theme by Vaga.

Why did I choose to use this new approach on publishing content?

Let me first talk about the drawbacks. I have to admit, you need some knowledge on how to do all this but once the setup is done, it’s only a different way of working. There are a lot of tutorials on the interwebs so you should be able to set this up by yourself by using your eyes and your common sense.

On the other hand, you have a website that is safe from unwanted access, it is blazing fast and uses a modern technology. That’s important for geeks like me. Next to all these pro’s I haven’t mentioned that you pay allmost nothing for your website. The only cost I have is the domain that I’ve bought at Cloudflare for less then 10 dollar.

Also my visitors don’t have to worry about their privacy. No data get’s collected, no cookies, no nothing. This website just serves you the information you need without any data harvesting. You’re welcome!
